# GitHub Growth Playbook

Public growth plan for Denis Redzic's GitHub profile and open reference repos.

## Thesis

These repos should earn stars because they turn painful builder problems into small, readable tools:

- AI voice transcript analytics.
- AI receptionist workflow logic.
- Receipt-backed QA for voice agents.
- Database reactivation campaign logic.
- Programmatic local SEO generation.
- Compliance-first lead routing.
- Ad creative intelligence.

The angle: useful reference engines for builders who want to inspect the implementation.

## What gets followers

- Working tools, not screenshots.
- Fictional examples people can run locally.
- A clear first 30 seconds: what it does, who it is for, how to run it, what output they get.
- Technical lessons from real engineering work.
- Consistent positioning across README, profile, portfolio, launch posts, and issues.

## Launch sequence

1. Pin the strongest repos on GitHub.
2. Set repo topics from each repo's launch pack.
3. Add one terminal screenshot or GIF per repo using fictional data only.
4. Post one repo per day.
5. Lead each post with the pain, then show the output, then link the repo.
6. Answer technical comments with implementation detail.
7. Add one small example or improvement every 2 to 3 days for two weeks.

## Weekly loop

- Monday: fix one README gap from a stranger's perspective.
- Tuesday: add one fictional example.
- Wednesday: share one technical lesson.
- Thursday: improve one CLI output or error message.
- Friday: ship one small feature and post a short changelog.
